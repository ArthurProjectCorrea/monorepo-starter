# 🧠 Boas Práticas e Dicas

## Organização
- Separe responsabilidades: apps para produtos finais, packages para recursos compartilhados.
- Documente cada pacote/app com README próprio.

## Padronização
- Use sempre as configs globais de ESLint, Prettier, tsconfig e Tailwind.
- Mantenha dependências atualizadas e evite duplicidade.

## Fluxo de trabalho
- Use `pnpm dev` para desenvolvimento integrado.
- Faça PRs pequenas e bem descritas.
- Rode `pnpm lint` e `pnpm test` antes de commitar.

## Customização
- Adapte scripts e configs conforme a necessidade do seu time/projeto.
- Use os utilitários e componentes compartilhados para acelerar o desenvolvimento.

## Dúvidas?
- Consulte a documentação em cada pasta.
- Veja exemplos nos scripts e configs.
- Abra issues ou PRs para melhorias!

> Um monorepo bem documentado e padronizado facilita a colaboração e a escalabilidade.
